Frequently Asked Questions About READMEs

What is a README?

+

A README is a text file that introduces and explains a project. It contains information that is commonly required to understand what the project is about.

How do I read the README?

+

You start at the top and move your eyes from left to right, then move down to the next line and repeat. Just kidding! Simply open the README file in your preferred text editor or viewer and peruse its contents.

What happens if I don't read the README?

+

The README police will come knocking at your door! Jokes aside, you might miss crucial information about the project, how to set it up, or important warnings.

Why is it called a README?

+

Because "PLEASE_FOR_THE_LOVE_OF_CODE_READ_THIS" was too long. The name is simply an imperative sentence: "Read me!"

How long should a README be?

+

As long as it needs to be, but not so long that it becomes a TLDR (Too Long; Didn't Read). Aim for comprehensive yet concise.
